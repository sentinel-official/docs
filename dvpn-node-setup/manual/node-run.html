Run the Node

Important

Before starting the node, remember to send a few P2P (50 should suffice) to the operator address otherwise the node will not start! To accomplish this, import the seed you generated or recovered in this step into a wallet such as Leap or Keplr.

If you are running your node for testing purposes you can use the Node Faucet

Run the Node Container​

Start your node by running:

docker run -d \
  --rm \
  --cap-drop ALL \
  --cap-add NET_ADMIN \
  --cap-add NET_RAW \
  --cap-add SYS_MODULE \
  --device /dev/net/tun \
  --sysctl net.ipv4.ip_forward=1 \
  --sysctl net.ipv6.conf.all.disable_ipv6=0 \
  --sysctl net.ipv6.conf.all.forwarding=1 \
  --sysctl net.ipv6.conf.default.forwarding=1 \
  --volume "${VOLUME}" \
  --name "dvpnx" \
  --interactive \
  --tty \
  --sig-proxy=false \
  ${PUBLISH_PORT_ARGS} \
  sentinel-dvpnx start

After Starting the Node​

Check if the node is running and visible to everyone. Open your browser and type the following URL

https://ip_node:tcp_port

Node logs. You can specify a different number of logs if you need to view more or fewer entries

docker logs -f -n 100 dvpnx

docker stop dvpnx

Start a stopped node

docker start dvpnx

docker rm -f dvpnx
